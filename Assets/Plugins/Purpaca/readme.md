# 嘿，注意！ 
为了保持这个项目的独立性，在此项目后续开发过程中，会编写此项目独属的音频管理系统取代此插件。现在使用这个插件只为临时解决一些需要通过代码直接播放音频或序列播放 `AudioClip` 的需求，所以在编写和音频相关的代码时，不要过度依赖此库！

### 如何避免过度依赖呢？
1.可以放心使用 `AudioManager.Play()`、`AudioManager.Pause()`、`AudioManager.Stop()` 这样的方法，我们不会太过改变此类方法的调用方式和内部实现原理。但当此插件被完全从项目中移除后，你仍需要清理命名空间引用或者轻微改变调用时使用的类名、方法名以及方法参数等内容。  

2.除了基本音频播放相关的方法，插件提供的其它方法都有后期被修改的风险，谨慎使用！  

3.音乐系统目前的部分上层实现依赖插件，后期内部实现需要改变，但外部调用不会发生改变

4.摆放在场景上的AudioSource不要直接连接到此插件内置的AudioMixer上！随着插件从项目中被移除，所有连接了插件内部AudioMixer上的AudioSource的配置都将失效并且需要重新设置。这里建议在前期开发时，遇到场景上的AudioSource需要接入音量管理时，额外创建代理AudioMixer后，先将AudioSource连接到代理AudioMixer后，再将代理AudioMixer连接到插件的内置AudioMixer上，这样可以大幅减少后期需要重新设置的内容的数量级（n -> 1）。**直接通过代码播放的音频不受此影响！**  

5.音频播放结束回调功能相关的，建议目前只在回调不需要后续处理（执行了就行，不用回收和管理）的情况下使用。
